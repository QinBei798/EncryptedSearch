# EncryptedSearch (加密全文搜索引擎原型)

**EncryptedSearch** 是一个基于 C++17 实现的加密全文搜索引擎原型工程。

本项目的核心设计理念为：**“先存密文，再搜明文”**。即文件以密文形态安全存储，系统能够对其内容关键词进行检索，且**搜索过程全程无需解密原文件**。项目集成了中国国家密码标准算法（国密 SM3/SM4），适用于对数据隐私有较高要求的文档检索工程验证与技术学习。

---

## ✨ 核心特性

### 🔒 数据安全与密码学实现 (Security)
- **SM4-CBC 加密存储**：原始文档使用 SM4 算法加密存储。每次加密自动生成随机 16 字节 IV 并写入密文头部，有效抵御模式攻击。
- **PBKDF2 密钥派生**：通过 `PBKDF2-SM3-HMAC` 算法，基于用户密码、随机盐 (Salt) 和迭代次数动态派生 SM4 密钥，避免源码硬编码密钥带来的安全风险。
- **哈希匹配搜索**：搜索阶段仅比对关键词的 SM3 哈希值，查询引擎本身不接触原始明文数据。

### ⚡ 索引与并发架构 (Indexing & Concurrency)
- **并行倒排索引构建**：采用生产者-消费者线程池模型，以文件为任务粒度进行批量哈希计算，充分利用多核 CPU 性能，降低锁竞争。
- **二进制紧凑索引格式**：设计了自定义的 `ESIX` 格式二进制定长索引文件。词典区驻留内存，支持 $O(\log N)$ 复杂度的二分查找；倒排列表 (Posting List) 存储于磁盘，通过偏移量按需随机读取。
- **流式分词与内存控制**：引入 64KB 块边界感知的流式读取机制，处理多字节 UTF-8 字符截断问题，支持大文件解析而不引起内存溢出 (OOM)。

### 🔎 检索与打分引擎 (Search & Ranking)
- **中英混合分词**：内联集成 `cppjieba` 分词库，采用 `CutForSearch` 模式以兼顾召回率与分词精度。
- **TF-IDF 相关性排序**：底层 Posting List 存储了 TF（词频）权重，结合全局文档数量计算 IDF（逆文档频率），实现搜索结果按相关性降序排列。
- **布尔逻辑搜索**：内置抽象语法树 (AST) 与递归下降解析器，支持多关键词的 `AND` / `OR` / `NOT` 集合运算，底层采用双指针扫描保证 $O(N+M)$ 复杂度的高效合并。

## 🏗️ 系统架构图

```text
         原始文件目录
              │
              ▼
    ┌─────────────────────┐
    │  cppjieba 流式分词  │  (64KB 边界感知分块解析)
    └─────────────────────┘
              │ 关键词列表
              ▼
    ┌─────────────────────┐
    │   ParallelHasher    │  (线程池，批量并行 SM3 哈希)
    └─────────────────────┘
              │ SM3(keyword) → DocID 倒排索引
              ▼
    ┌─────────────────────┐
    │     IndexManager    │  写入二进制索引 encrypted_search_index.bin
    └─────────────────────┘

* 索引构建阶段：原始文件 ──SM4-CBC(随机IV)──▶ cipher_docs/{docId}.cipher （密文存储区）
* 搜索查询阶段：用户关键词 ──SM3──▶ 哈希值 ──二分查找──▶ Posting List(附带TF) ──▶ TF-IDF 排序返回
```

## 🚀 编译与运行

### 前置依赖
- **CMake** (>= 3.15)
- **C++17 编译器** (GCC 8+ / MSVC 2019+ / Clang)
- **GmSSL** (国密算法库，需提前安装并在系统路径中)
*(注：cppjieba 分词库已包含在 `third_party` 中作为源码级依赖，开箱即用)*

### 构建步骤

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/EncryptedSearch.git
cd EncryptedSearch

# 2. 创建构建目录
mkdir build && cd build

# 3. 运行 CMake
cmake ..

# 4. 编译项目
cmake --build . --config Release
```
*(编译完成后，分词所需的字典文件会自动拷贝到生成目录)*

## 🗺️ 开发路线图 (Roadmap)

本项目按照从易到难的顺序进行持续迭代，当前进度如下：

- [x] **Phase 1: 随机 IV 加密** (安全) - 消除固定 IV，防止密文模式泄露。
- [x] **Phase 2: 任务批量化** (性能) - 优化并行哈希任务粒度，大幅减少线程锁竞争。
- [x] **Phase 3: 流式分词** (健壮) - 解决单次全量读文件导致的内存崩溃，支持超大文件。
- [x] **Phase 4: PBKDF2 密钥派生** (安全) - 消除源码硬编码密钥，实现基于用户密码的安全派生。
- [x] **Phase 5: TF-IDF 排序** (功能) - 升级二进制索引格式，支持相关性智能打分排序。
- [x] **Phase 6: 布尔搜索 (Boolean Search)** (功能) - 支持多关键词的 `AND` / `OR` / `NOT` 语法集合运算。
