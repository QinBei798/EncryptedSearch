# EncryptedSearch (加密全文搜索引擎)

**EncryptedSearch** 是一个基于 C++17 实现的高性能**加密全文搜索引擎原型**。

本项目的核心设计理念是：**“先存密文，再搜明文”**。即文件以密文形态静态安全存储，但系统仍能对其内容关键词进行快速检索，且**搜索过程绝对不解密原文件**。项目深度集成了中国国家密码标准算法（国密 SM3/SM4），适用于对数据隐私和安全性要求极高的文档检索场景。

---

## ✨ 核心特性

### 🔒 军工级安全隔离 (Security)
- **SM4-CBC 加密存储**：原文档使用 SM4 算法加密，每次加密自动生成随机 IV 写入文件头，抵御模式攻击。
- **PBKDF2 密钥派生**：彻底摒弃硬编码密钥，基于用户密码、随机盐 (Salt) 和迭代，通过 `PBKDF2-SM3-HMAC` 动态派生 SM4 密钥。
- **零知识搜索**：搜索过程仅比对关键词的 SM3 哈希值，全程不接触原始明文数据。

### ⚡ 高性能架构 (Performance)
- **并行倒排索引 (Parallel Indexing)**：采用生产者-消费者线程池模型，以文件为粒度批量并行计算 SM3 哈希。
- **二进制紧凑索引**：自定义高性能 `ESIX` 格式二进制索引文件，字典区支持 $O(\log N)$ 复杂度二分查找，Posting List 支持快速偏移读取。
- **内存极低占用**：引入 64KB 块边界感知的**流式分词**，支持 GB 级超大日志文件的无损解析，避免 OOM 崩溃。

### 🧠 智能检索 (Smart Retrieval)
- **中英文混合分词**：内联集成 `cppjieba`，采用 `CutForSearch` 模式兼顾最大召回率与分词精度。
- **TF-IDF 相关性排序**：不仅支持“是否存在”，还在底层 Posting List 融入 TF（词频）权重，结合全局 IDF 计算评分，让最匹配的文档排在最前面。

## 🏗️ 系统架构图

```text
         原始文件目录
              │
              ▼
    ┌─────────────────────┐
    │  cppjieba 流式分词  │  (64KB 边界感知分块解析)
    └─────────────────────┘
              │ 关键词列表
              ▼
    ┌─────────────────────┐
    │   ParallelHasher    │  (线程池，批量并行 SM3 哈希)
    └─────────────────────┘
              │ SM3(keyword) → DocID 倒排索引
              ▼
    ┌─────────────────────┐
    │      IndexManager   │  写入二进制索引 encrypted_search_index.bin
    └─────────────────────┘

* 并行执行：原始文件 ──SM4-CBC(随机IV)──▶ cipher_docs/{docId}.cipher （密文保险箱）
* 搜索阶段：用户关键词 ──SM3──▶ 哈希 ──二分查找──▶ Posting List(带有TF) ──▶ TF-IDF 排序返回
```

## 🚀 编译与运行

### 前置依赖
- **CMake** (>= 3.15)
- **C++17 编译器** (GCC 8+ / MSVC 2019+ / Clang)
- **GmSSL** (国密算法库，需提前安装并在系统路径中)
*(注：cppjieba 分词库已包含在 `third_party` 中作为源码级依赖，开箱即用)*

### 构建步骤

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/EncryptedSearch.git
cd EncryptedSearch

# 2. 创建构建目录
mkdir build && cd build

# 3. 运行 CMake
cmake ..

# 4. 编译项目
cmake --build . --config Release
```
*(编译完成后，分词所需的字典文件会自动拷贝到生成目录)*

## 🗺️ 开发路线图 (Roadmap)

本项目按照从易到难的顺序进行持续迭代，当前进度如下：

- [x] **Phase 1: 随机 IV 加密** (安全) - 消除固定 IV，防止密文模式泄露。
- [x] **Phase 2: 任务批量化** (性能) - 优化并行哈希任务粒度，大幅减少线程锁竞争。
- [x] **Phase 3: 流式分词** (健壮) - 解决单次全量读文件导致的内存崩溃，支持超大文件。
- [x] **Phase 4: PBKDF2 密钥派生** (安全) - 消除源码硬编码密钥，实现基于用户密码的安全派生。
- [x] **Phase 5: TF-IDF 排序** (功能) - 升级二进制索引格式，支持相关性智能打分排序。
- [ ] **Phase 6: 布尔搜索 (Boolean Search)** (功能) - **[🚧 研发中]** 支持多关键词的 `AND` / `OR` / `NOT` 语法集合运算。
